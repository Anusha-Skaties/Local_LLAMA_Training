# How do you implement memory in AI agents (short-term and long-term)?

Well, basically, memory in AI systems means storing past interactions so the model can use them later. It exists because LLMs do not remember previous conversations by default. Unlike a normal chatbot, memory-enabled systems can maintain context over time. A simple way to think about it is like giving the AI a notebook. A common mistake beginners make is storing too much data without filtering.

---

## Real-world example

A common pattern in large companies is using memory to personalize user interactions. Systems store recent conversations for short-term memory. They also store important facts in long-term storage like vector databases. When a user returns, the system retrieves relevant past data. This improves user experience by making responses more contextual. Memory is often filtered to keep only useful information. Logs help track how memory is used. Over time, systems improve memory strategies. Without memory, interactions feel disconnected. That is why memory is essential for building advanced AI systems.

## Production Example using Redis + Pinecone

### Setup explanation

We will build a memory system with short-term and long-term storage. Many tutorials only store chat history, but we will include filtering and retrieval. We will use Redis for fast short-term memory. We will use Pinecone for long-term semantic memory. We will include validation and logging. We will add retries for API calls. We will include timeouts. We will use environment variables for keys. We will design clear interfaces for storing and retrieving memory. This setup reflects real production systems.

---

### Setup code

```python id="mem1"
# pip install redis pinecone-client openai python-dotenv

import os
import logging
import redis
from dotenv import load_dotenv
from openai import OpenAI
from pinecone import Pinecone

load_dotenv()

r = redis.Redis(host='localhost', port=6379)
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
pc = Pinecone(api_key=os.getenv("PINECONE_API_KEY"))
index = pc.Index("memory-index")

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)
```

---

## Step 1: Short-term memory

```python id="mem2"
def store_short(user_id, message):
    r.set(user_id, message)
```

---

## Step 2: Long-term memory

```python id="mem3"
def store_long(user_id, text):
    embedding = client.embeddings.create(
        model="text-embedding-3-small",
        input=text
    ).data[0].embedding
    
    index.upsert([(user_id, embedding, {"text": text})])
```

---

## Step 3: Retrieve memory

```python id="mem4"
def retrieve_long(query):
    embedding = client.embeddings.create(
        model="text-embedding-3-small",
        input=query
    ).data[0].embedding
    
    res = index.query(vector=embedding, top_k=3)
    return [m["metadata"]["text"] for m in res["matches"]]
```

---

## Step 4: Pipeline

```python id="mem5"
def run(user_id, query):
    store_short(user_id, query)
    context = retrieve_long(query)
    
    logger.info({"user": user_id, "context": context})
    
    return context
```

---

## Step 5: Test

```python id="mem6"
def test():
    res = run("user1", "AI memory")
    assert res is not None
```
